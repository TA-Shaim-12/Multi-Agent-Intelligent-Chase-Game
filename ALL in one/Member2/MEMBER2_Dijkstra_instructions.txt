TASK 1 — Add comments to dijkstra_path function
  Find the function called dijkstra_path
  Add a comment above every line inside it explaining what it does
  Key things to mention in your comments:
    - Dijkstra uses terrain cost: MUD costs 3, STONE costs 2, empty costs 1
    - It uses a priority queue (heap) to always pick the cheapest path first
    - It finds the cheapest cost path, not the shortest number of steps
    - Unlike BFS it avoids mud and stone tiles because they cost more

TASK 2 — Add comments to greedy_path function
  Find the function called greedy_path
  Add a comment above every line inside it explaining what it does
  Key things to mention in your comments:
    - Greedy only looks at distance to the goal, ignores terrain cost
    - It is the fastest algorithm but NOT always optimal
    - It can fail or take bad routes on maps with walls
    - It uses Manhattan distance: abs(row difference) + abs(col difference)

HOW DIJKSTRA IS DIFFERENT FROM BFS
------------------------------------
BFS        : counts steps only, ignores terrain, uses a regular queue
Dijkstra   : counts real cost (mud=3, stone=2), uses a priority queue
Greedy     : only looks at distance to goal, fastest but not optimal
