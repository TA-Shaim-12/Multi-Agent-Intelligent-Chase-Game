  Phase 1 — Backward Dijkstra:
    The algorithm starts from the GOAL and works backwards
    It calculates the exact terrain cost from every nearby cell
    back to the goal, within a radius of 18 cells
    The result is stored in a dictionary called bdist

  Phase 2 — Forward A*:
    Instead of Manhattan distance as the heuristic h(n),
    it uses the exact cost from bdist when available
    This is more accurate because it knows real terrain costs
    Outside the 18-cell radius it falls back to Manhattan distance

  Why it is still optimal:
    h_adv never overestimates the true cost
    Inside radius: h_adv = exact cost = true cost (never overestimates)
    Outside radius: h_adv = Manhattan, which also never overestimates
    So Advanced A* finds the optimal path every time

WHAT TO DO IF THERE IS A MERGE CONFLICT
-----------------------------------------
A conflict happens when two members edited the same lines.
Git marks the file like this:

  <<<<<<< HEAD
      # this is the version already in main
  =======
      # this is the version from the branch
  >>>>>>> bfs_S12345

To fix it:
  1. Open ai_chase_game_final.py
  2. Find the lines with <<<<<<< and >>>>>>>
  3. Keep the correct version (usually the branch version)
  4. Delete ALL three marker lines
     (the <<<<<<<  line, the =======  line, and the >>>>>>>  line)
  5. Save the file
  6. Run:
       git add ai_chase_game_final.py
       git commit -m "merge: fix conflict in bfs section"
       git push origin main

To prevent conflicts in the future:
  Remind everyone to only edit their assigned lines.
  Conflicts only happen when two people edit the same lines.
